# specs (cross-cutting)

Design specs for work that spans more than one package (e.g. a new course lesson's
feature). One file per spec: the problem, the proposed shape, and the contract
changes. Module-local specs go in `<module>/specs/`.

Canonical *current* behavior is the root [`README.md`](../README.md) and
[`docs/architecture.md`](../docs/architecture.md); specs here describe *intended*
or *in-progress* change, not the shipped system.
